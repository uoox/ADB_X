# ADB_X v1.2.2

## Critical fix: module was undetectable by LSPosed
- The 1.2.0 migration to the libxposed API dropped the legacy
  `xposedmodule` manifest meta-data, keeping only the
  `META-INF/xposed/` declaration. LSPosed still discovers modules
  primarily from that manifest marker, so ADB_X stopped appearing in
  the LSPosed manager entirely — it could never be ticked or enabled,
  and always reported "module not active". Restored the
  `xposedmodule` / `xposeddescription` / `xposedminversion` /
  `xposedscope` meta-data (with a matching `@array/xposed_scope`)
  alongside the modern `META-INF/xposed/` entry, so the module is both
  detected by every LSPosed and loaded through the modern
  `java_init.list` entry point.

# ADB_X v1.2.1

## Bug fixes
- Fixed a duplicated `when` branch in the bottom-navigation handler
  (`R.id.tab_networks` was listed twice) — harmless but dead.
- `PairingReceiver` now actually persists the captured pairing port/code:
  it wrote to `/data/local/tmp` with a plain app-uid file write (which
  always fails there) despite its own doc promising an elevated `su`
  write. It now falls back to `ShellUtils.executeSu`, matching the rest
  of the app.

## Feature completion
- Wired up the boot-time KernelSU/Magisk module installer. The
  `installAsKernelSuModule()` path shipped in 1.2.0 but was never
  invoked; it now runs once per app version on a background thread
  (root-gated, idempotent) so the trusted-Wi-Fi rule can fire at boot
  before the lockscreen, where OnePlus throttles background broadcasts.

## Cleanup
- Removed the dead `TrustedWifiWatcher` (superseded by
  `WifiStateReceiver`), the never-called `MainActivity.pollPairingPort()`
  and `WifiAdapter.refresh()`, and dead null/context guards.
- Removed 37 leftover `TODO: document` placeholder KDoc blocks that the
  1.2.0 notes claimed were already gone.
- Corrected stale doc references to the removed `handleLoadPackage`
  callback, fixed duplicated comment numbering, and de-duplicated the
  `ACCESS_NETWORK_STATE` / `ACCESS_WIFI_STATE` manifest permissions.
- Replaced the placeholder unit test with real `WifiHelper.cleanSsid`
  coverage.

# ADB_X v1.2.0

## Migrated to the modern libxposed API (102)
- Replaced the legacy `de.robv.android.xposed:api:82` framework with
  `io.github.libxposed:api:102.0.0`.
- Entry point moved from `assets/xposed_init` to
  `META-INF/xposed/java_init.list`; scope and API version now live in
  `META-INF/xposed/scope.list` and `META-INF/xposed/module.prop`.
- All method/constructor hooks rewritten to the interceptor-chain model
  (`XposedInterface.hook(...).intercept { ... }`); every `XposedHelpers`
  call replaced with plain reflection.

## Fixes & improvements
- Pairing-dialog hooks now hook every declared constructor instead of
  guessing a single `(Object)` signature, so OEM builds that added
  constructor parameters still surface the pairing port.
- Pairing-port marker is written directly first and only falls back to an
  elevated shell write, avoiding a redundant `su` spawn on every capture.
- Removed dead code (unused context lookup in the port writer) and the
  placeholder "TODO: document" comments.
- Added the missing JUnit test dependency so the unit tests compile.
- Dropped the now-unused `api.xposed.info` Maven repo and legacy Xposed
  manifest meta-data.
